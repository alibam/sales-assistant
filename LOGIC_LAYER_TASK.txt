## 三大任务

### 任务 1：编写安全的数据查询层 (Server Actions) ⚠️ 红线级别

**文件位置：**
- `app/(dashboard)/actions/stats.ts`

**必须实现的方法：**

1. `getCustomerStats()` - 获取客户统计
   - 返回 A/B/C/D 四个等级的客户数量
   - 必须按 tenantId 过滤
   
2. `getFollowUpCustomers()` - 获取待跟进客户列表
   - 返回 B/C 级客户（需要跟进）
   - 按 updatedAt 倒序排列
   - 限制返回 10 条
   - 必须按 tenantId 过滤

3. `getSalesFunnel()` - 获取销售漏斗数据（Manager 用）
   - 返回各阶段客户数量和转化率
   - 必须按 tenantId 过滤

4. `getAlerts()` - 获取异常卡点警告（Manager 用）
   - 返回长时间未跟进的客户
   - 必须按 tenantId 过滤

**🚨 绝对红线（违反即回滚）：**
```typescript
// ❌ 绝对禁止！没有 tenantId 过滤的查询
await prisma.customer.findMany({
  where: { status: 'A' }  // 危险！会泄露其他租户数据
});

// ✅ 正确！必须包含 tenantId
await prisma.customer.findMany({
  where: { 
    tenantId: session.tenantId,  // 必须！
    status: 'A' 
  }
});
```

**类型安全要求：**
- 所有函数必须有明确的返回类型
- 使用 Prisma 生成的类型，不要用 `any`
- 错误处理必须完善（try-catch）
