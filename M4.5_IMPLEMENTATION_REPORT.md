### ✅ 任务 2：实现双轨交互分流

#### 2.1 ✅ 创建双轨引擎核心逻辑

**新建文件**: `lib/ai/dual-track-engine.ts`

**核心接口**:
- `DualTrackInput`: 输入参数（mode, customerProfile, userInput, conversationHistory）
- `DualTrackOutput`: 输出结果（aiResponse, completionRate, missingFields, nextSteps 等）

**核心函数**:
1. ✅ `processDualTrack()` - 主入口，根据 mode 分发
2. ✅ `processCopilotMode()` - 实时引导模式
   - 调用 `runGapAnalysis()` 提取信息并计算完成度
   - 如果完成度 < 80%，调用 `generateCustomerQuestion()` 生成给客户的话术
   - 如果完成度 >= 80%，返回完成信号
3. ✅ `processPostCallMode()` - 事后复盘模式
   - 调用 `runGapAnalysis()` 提取信息并计算完成度
   - 如果完成度 < 80%，调用 `generateSalesQuestion()` 生成对销售的追问
   - 如果完成度 >= 80%，返回完成信号
4. ✅ `generateCustomerQuestion()` - 使用 AI 生成给客户的话术
5. ✅ `generateSalesQuestion()` - 使用 AI 生成对销售的追问

### ✅ 任务 3：数据库 Schema 更新

**文件**: `prisma/schema.prisma`

**新增模型**: `ConversationHistory`
```prisma
model ConversationHistory {
  id         String   @id @default(uuid()) @db.Uuid
  customerId String   @map("customer_id") @db.Uuid
  tenantId   String   @map("tenant_id") @db.Uuid
  role       String   @db.VarChar(20)  // 'user' | 'assistant'
  content    String   @db.Text
  mode       String   @db.VarChar(20)  // 'copilot' | 'postCall'
  createdAt  DateTime @default(now()) @map("created_at")

  customer   Customer @relation(fields: [customerId], references: [id], onDelete: Cascade)

  @@index([customerId, tenantId], map: "idx_conversation_customer_tenant")
  @@index([tenantId], map: "idx_conversation_tenant_id")
  @@map("conversation_history")
}
```

**更新 Customer 模型**:

## 技术亮点

1. ✅ **加权完成度算法**：根据字段重要性计算画像完成度，而非简单的字段数量统计
2. ✅ **双轨分流架构**：统一的引擎接口，根据模式动态生成不同的话术
3. ✅ **AI 驱动的话术生成**：使用 `generateText` 生成自然、专业的提问话术
4. ✅ **严格的租户隔离**：所有数据库操作都包含 tenantId 过滤
5. ✅ **流式 UI 反馈**：进度条实时更新，给销售员极强的正向反馈
6. ✅ **对话历史追踪**：记录完整的交互历史，支持上下文理解
