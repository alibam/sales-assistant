## 三大审查点（必须全部通过）

### 审查点 1：多租户与数据安全（绝对红线）

**检查项 1.1：ConversationHistory Schema 定义**

文件：`prisma/schema.prisma`

必须验证：
- ✅ ConversationHistory 模型是否包含 `tenantId` 字段？
- ✅ `tenantId` 字段是否正确映射到数据库列（`@map("tenant_id")`）？
- ✅ 是否有正确的索引（`@@index([customerId, tenantId])`）？
- ✅ 是否有租户隔离索引（`@@index([tenantId])`）？

**检查项 1.2：handleFollowUp() 租户隔离**

文件：`app/(dashboard)/customer-demo/actions.ts`

必须验证：
- ✅ 查询客户时，是否包含 `where: { id: customerId, tenantId: session.tenantId }`？
- ✅ 查询对话历史时，是否包含 `where: { customerId, tenantId: session.tenantId }`？
- ✅ 插入对话记录（user）时，是否包含 `tenantId: session.tenantId`？
- ✅ 插入对话记录（assistant）时，是否包含 `tenantId: session.tenantId`？
- ✅ 更新客户画像时，是否验证了租户归属？

**检查项 1.3：双轨引擎租户隔离**

文件：`lib/ai/dual-track-engine.ts`

必须验证：
- ✅ 所有数据库查询是否都通过 Server Action 进行（不直接访问 Prisma）？
- ✅ 如果有直接查询，是否包含 `tenantId` 过滤？

**红线标准**：
- 任何一个数据库查询缺少 `tenantId` 过滤 → **立即打回，要求 Claude Code 修复**
- 任何一个插入操作缺少 `tenantId` 字段 → **立即打回，要求 Claude Code 修复**
