   ```typescript
       tenantId,  // ✅ 正确
       role: 'assistant',

   ```

#### ❌ CRITICAL 问题：

**问题位置**：Line 172-179

```typescript
// 5. 更新客户画像（租户隔离）
if (result.updatedProfile) {
  await prisma.customer.update({
    where: { id: customerId },  // ❌ 缺少 tenantId 验证！
    data: {
      profileData: result.updatedProfile as any,
      updatedAt: new Date(),
    },
  });
}
```

**问题描述**：
- 更新客户画像时，`where` 子句只包含 `id: customerId`
- **没有验证 `tenantId`**，违反了租户隔离原则
- 虽然前面已经查询过客户并验证了租户归属，但 Prisma 的 `update` 操作是独立的数据库查询
- 如果在查询和更新之间，客户被转移到其他租户（虽然不太可能），或者代码逻辑被修改，这将导致跨租户数据泄露

**安全影响**：
- **严重程度**：CRITICAL
- **攻击场景**：恶意用户可能通过竞态条件或代码修改，绕过前置检查，更新其他租户的客户数据
- **数据泄露风险**：高 - 可能导致跨租户数据污染

**修复建议**：
```typescript
// 5. 更新客户画像（租户隔离）
if (result.updatedProfile) {
  await prisma.customer.update({
    where: {
      id: customerId,
      tenantId: tenantId,  // ✅ 必须添加
    },
    data: {
      profileData: result.updatedProfile as any,
      updatedAt: new Date(),
    },
  });
}
```

**结论**：✅ **通过**（修复后）

**修复记录**：
- 原问题：Line 174 缺少 `tenantId` 验证
- 修复时间：2026-03-05
- 修复内容：在 `where` 子句中添加 `tenantId: tenantId`
- 验证状态：✅ 已验证通过
