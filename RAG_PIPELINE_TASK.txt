### 任务 1：编写 Document Processor (文档处理器)

**文件位置**：`lib/ai/document-processor.ts`

**核心功能**：文本切片（Chunking）

**实现要求**：
```typescript
/**
 * 将长文本切分为小块（Chunk）
 * 
 * 策略：
 * 1. 优先按段落切分（\n\n）
 * 2. 如果段落过长，按固定字数切分（如 500 字）
 * 3. 保留一定的重叠（overlap），避免语义断裂
 * 
 * @param text - 原始文本
 * @param options - 切片选项
 * @returns 文本块数组
 */
export function chunkText(
  text: string,
  options?: {
    maxChunkSize?: number;  // 默认 500
    overlap?: number;        // 默认 50
  }
): string[];
```

**测试用例**：
- 短文本（< 500 字）：返回单个块
- 长文本（> 500 字）：按段落或固定字数切分
- 超长段落：强制切分并保留重叠

### 任务 2：对接 Qwen3-Embedding 端点

**文件位置**：`lib/ai/embedding.ts`（新建）

**环境变量**：
- `EMBEDDING_API_URL` - Qwen3-Embedding 端点地址
- `EMBEDDING_API_KEY` - API 密钥
- `EMBEDDING_MODEL` - 模型名称（如 "qwen3-embedding"）

**核心功能**：
```typescript
/**
 * 调用 Qwen3-Embedding API 获取文本向量
 * 
 * @param text - 输入文本
 * @returns 1024 维向量数组
 */
export async function getEmbedding(text: string): Promise<number[]> {
  const response = await fetch(process.env.EMBEDDING_API_URL!, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${process.env.EMBEDDING_API_KEY}`,
    },
    body: JSON.stringify({
      model: process.env.EMBEDDING_MODEL || 'qwen3-embedding',
      input: text,
    }),
  });
  
  const data = await response.json();
  return data.data[0].embedding; // 返回 1024 维向量
}
```

**错误处理**：
- API 调用失败时抛出明确的错误
- 验证返回的向量维度是否为 1024

### 任务 3：创建知识库上传 Server Action

**文件位置**：`app/(dashboard)/actions/knowledge.ts`（新建）

**核心 Server Action**：
```typescript
'use server';

import { requireAuth } from '@/lib/auth/session';
import { prisma } from '@/lib/db/client';
import { chunkText } from '@/lib/ai/document-processor';
import { getEmbedding } from '@/lib/ai/embedding';

/**
 * 上传并向量化文档
 * 
 * 流水线：
 * 1. 验证用户身份，获取 tenantId
 * 2. 在 KnowledgeBase 表创建记录
 * 3. 切分文本为多个块
 * 4. 为每个块调用 Embedding API
 * 5. 将块和向量存入 DocumentChunk 表
 * 
 * @param title - 文档标题
 * @param content - 文档内容（纯文本）
 * @returns 成功消息或错误
 */
export async function uploadDocument(
  title: string,
  content: string
): Promise<{ success: boolean; message: string; kbId?: string }> {
  try {
    // 1. 验证身份
    const session = await requireAuth();
    
    // 2. 创建知识库记录
    const kb = await prisma.knowledgeBase.create({
      data: {
        tenantId: session.tenantId,
        name: title,
        description: `上传于 ${new Date().toLocaleString()}`,
      },
    });
    
    // 3. 切分文本
    const chunks = chunkText(content);
    
    // 4. 批量向量化并入库
    for (const chunk of chunks) {
      const embedding = await getEmbedding(chunk);
      
      // 5. 插入 DocumentChunk（使用 $executeRaw 处理 vector 类型）
      await prisma.$executeRaw`
        INSERT INTO document_chunks (
          id, tenant_id, knowledge_base_id, content, embedding, created_at, updated_at
        ) VALUES (
          gen_random_uuid(),
          ${session.tenantId}::uuid,
          ${kb.id}::uuid,
          ${chunk},
          ${embedding}::vector(1024),
          NOW(),
          NOW()
        )
      `;
    }
    
    return {
      success: true,
      message: `成功上传文档，共 ${chunks.length} 个文本块`,
      kbId: kb.id,
    };
  } catch (error) {
    console.error('[Upload Document] Error:', error);
    return {
      success: false,
      message: error instanceof Error ? error.message : '上传失败',
    };
  }
}
```

**关键技术点**：
1. **Prisma vector 插入**：必须使用 `$executeRaw` 或 `$queryRaw`，因为 Prisma 不直接支持 `Unsupported` 类型的写入
2. **向量格式**：PostgreSQL 的 pgvector 需要 `[1,2,3,...]::vector(1024)` 格式
3. **租户隔离**：所有记录都必须包含 `tenantId`
4. **错误处理**：任何步骤失败都要回滚（考虑使用事务）

### 任务 4：管理员看板的上传入口

**文件位置**：`app/(dashboard)/components/AdminDashboard.tsx`

**UI 改造**：
```typescript
'use client';

import { useState } from 'react';
import { uploadDocument } from '../actions/knowledge';

export default function AdminDashboard({ session }: Props) {
  const [title, setTitle] = useState('');
  const [content, setContent] = useState('');
  const [loading, setLoading] = useState(false);
  const [message, setMessage] = useState('');

  const handleUpload = async () => {
    if (!title || !content) {
      setMessage('请填写标题和内容');
      return;
    }
    
    setLoading(true);
    setMessage('');
    
    try {
      const result = await uploadDocument(title, content);
      setMessage(result.message);
      
      if (result.success) {
        setTitle('');
        setContent('');
      }
    } catch (error) {
      setMessage('上传失败：' + (error as Error).message);
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="p-6">
      <h1 className="text-2xl font-bold mb-6">管理员看板</h1>

      {/* 知识库上传表单 */}
      <div className="bg-white p-6 rounded shadow">
        <h2 className="text-xl font-bold mb-4">知识库上传</h2>
        
        <div className="mb-4">
          <label className="block text-sm font-medium mb-2">文档标题</label>
          <input
            type="text"
            value={title}
            onChange={(e) => setTitle(e.target.value)}
            className="w-full border rounded px-3 py-2"
            placeholder="例如：产品手册 v1.0"
          />
        </div>
        
        <div className="mb-4">
          <label className="block text-sm font-medium mb-2">文档内容</label>
          <textarea
            value={content}
            onChange={(e) => setContent(e.target.value)}
            className="w-full border rounded px-3 py-2 h-64"
            placeholder="粘贴文档内容..."
          />
        </div>
        
        <button
          onClick={handleUpload}
          disabled={loading}
          className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600 disabled:bg-gray-400"
        >
          {loading ? '向量化中...' : '向量化并保存'}
        </button>
        
        {message && (
          <div className={`mt-4 p-3 rounded ${
            message.includes('成功') ? 'bg-green-100 text-green-800' : 'bg-red-100 text-red-800'
          }`}>
            {message}
          </div>
        )}
      </div>
    </div>
  );
}
```

**注意**：AdminDashboard 需要改为 Client Component（'use client'），因为需要使用 useState

## 关键技术难点

### 1. Prisma vector 类型插入

**问题**：Prisma 的 `Unsupported("vector(1024)")` 类型无法直接写入

**解决方案**：使用 `$executeRaw`
```typescript
await prisma.$executeRaw`
  INSERT INTO document_chunks (...)
  VALUES (..., ${embedding}::vector(1024), ...)
`;
```

**注意**：
- `embedding` 必须是 `number[]` 类型
- PostgreSQL 会自动将 JSON 数组转换为 vector 类型
- 必须显式指定 `::vector(1024)` 类型转换

### 2. 向量数组格式

**正确格式**：
```typescript
const embedding = [0.1, 0.2, 0.3, ...]; // 1024 个数字
```

**错误格式**：
```typescript
const embedding = "[0.1, 0.2, 0.3, ...]"; // ❌ 字符串
```

### 3. 事务处理（可选但推荐）

如果向量化过程中某个块失败，应该回滚整个操作：
```typescript
await prisma.$transaction(async (tx) => {
  const kb = await tx.knowledgeBase.create(...);
  
  for (const chunk of chunks) {
    const embedding = await getEmbedding(chunk);
    await tx.$executeRaw`INSERT INTO ...`;
  }
});
```
